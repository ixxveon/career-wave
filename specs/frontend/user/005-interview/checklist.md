## Phase 1 — 상태 관리 & 데이터 연동

### 상태 설계

- [ ] 서버 상태(리포트, 이력 목록)는 `TanStack Query`로 관리하고, 세션·실시간 UI 상태는 `useReducer`로 관리하는가?
- [ ] TanStack Query 캐시 데이터를 컴포넌트 `useState`에 복사하여 동기화를 깨뜨리지 않았는가?
- [ ] 상태 머신 전이(`READY → RUNNING → RECONNECTING → FINISHED / ERROR`)가 누락 없이 처리되는가?
- [ ] `useReducer` 외부에서 액션 없이 상태를 강제 변경하는 코드가 없는가?

### 데이터 연동

- [ ] 데이터 계약: `api-schema.md`에 정의된 JSON 구조와 실제 API 호출/응답 구조가 일치하는가?
- [ ] 세션 유지: 페이지 새로고침 시에도 `sessionId` 및 면접 상태가 유실되지 않는가?
  > sessionStorage 적용 (탭 종료 시 자동 만료 의도 — localStorage 사용 금지)
- [ ] 비정상 종료 후 재진입 시, `RUNNING` 상태가 `sessionStorage` 기반으로 복원되는가?

### 에러 핸들링

- [ ] API 호출 실패 시 `statusCode`에 따른 공통 모달/메시지가 정상 노출되는가?
- [ ] WebSocket `ERROR` 타입 메시지 수신 시 에러 상태 전이 및 UI 처리가 되는가?
- [ ] 네트워크 단절 상태에서 자동 재연결 시도 후 최대 횟수 초과 시 안내 메시지가 표시되는가?

## Phase 2 — 기능 구현 (P1: 핵심 사용자 플로우)

> P1: AI 면접의 핵심 플로우. 이 항목들이 동작하지 않으면 기능 자체가 불가.

### Pre-flight Check (사전 진단)

- [ ] 마이크 권한(`navigator.mediaDevices.getUserMedia`) 확인이 면접 진입 전에 수행되는가?
- [ ] WebSocket 서버 연결 가능 여부(ping/pong)가 면접 진입 전에 검증되는가?
- [ ] 사전 진단 실패 시 면접 시작 버튼이 비활성화되고 적절한 안내 메시지가 노출되는가?

### 텍스트 면접

- [ ] 텍스트 답변 입력 후 전송 시 채팅창에 정상 반영되는가?
- [ ] 전송 중 중복 요청이 방지되는가? (요청 중 버튼 비활성화)
- [ ] 미입력 상태에서 전송 시 차단 및 안내가 동작하는가?

### 음성 면접

- [ ] MediaRecorder가 정상 동작하며 5초 단위 Chunk로 분할 전송되는가?
- [ ] 마이크 권한 거부 시 에러 처리 및 텍스트 전환 안내가 노출되는가?
- [ ] Safari 등 WebM 미지원 브라우저에서 MIME 타입 분기 처리가 동작하는가?
- [ ] STT 변환 결과(`STT_RESULT`)가 실시간으로 채팅창에 반영되는가?
- [ ] STT 변환 실패 시 에러 토스트 노출 및 텍스트 직접 입력 전환 안내가 동작하는가?
- [ ] 컴포넌트 언마운트 시 `.getTracks().forEach(track => track.stop())`으로 스트림이 해제되는가?
- [ ] 면접 종료 시 혹은 탭 닫기 시 진행 중이던 모든 WebSocket 연결과 MediaStream이 정상적으로 dispose 되는가?
  > `window.beforeunload` 이벤트 또는 컴포넌트 언마운트 시 WebSocket `.close()` + MediaStream `.getTracks().forEach(track => track.stop())` 호출 여부 확인

### TTS 재생

- [ ] TTS 오디오 청크가 Queue에 담겨 순차 재생되는가? (이전 문장 완료 후 다음 재생)
- [ ] 오디오가 겹쳐서 재생되는 현상이 없는가?
- [ ] TTS 재생 실패 시 에러 토스트가 노출되고 면접 흐름이 유지되는가?

### 타이머

- [ ] 문항당 2분 30초(150초) 타이머가 정상 동작하는가?
- [ ] 타이머 만료 시 `MediaRecorder.stop()`이 즉시 호출되고 잔여 버퍼가 서버로 전송되는가?
- [ ] 타이머 임박(예: 30초 이하) 시 시각적 강조 표시가 있는가?

### 실시간 채팅창

- [ ] AI 질문(`QUESTION`)과 유저 답변이 순서에 맞게 채팅창에 렌더링되는가?
- [ ] `questionOrder`를 기준으로 메시지 시퀀스가 보장되는가?
- [ ] 스크립트 길이가 서버 설정 최대 질문 수를 초과할 경우 Virtual List 또는 스크롤 제어가 적용되는가?
- [ ] LLM 스트리밍 토큰이 `ref` + `requestAnimationFrame` 기반으로 부드럽게 렌더링되는가?

### 리포트 시각화

- [ ] Relevance·Depth·Delivery·Fluency 4개 지표 레이더 차트가 정상 렌더링되는가?
- [ ] 음성 품질 50% 미만 또는 텍스트 면접 문항의 Delivery·Fluency 점수가 `null` 처리되고 마스킹 안내 UI가 노출되는가?
- [ ] 문항별 AI 피드백(`aiFeedback`)이 스크립트와 함께 정상 표시되는가?
- [ ] 불필요한 리렌더링이 없는가? (React DevTools Profiler로 확인, `memo`·`useMemo` 적용)

## Phase 3 — 연동 계약 (P2: 인접 도메인 연결)

> P2: 단독 기능은 동작하지만, 타 도메인과의 연결이 필요한 항목.

### 서류 도메인 연동

- [ ] `?documentId=xxx` 쿼리 파라미터가 정상적으로 파싱되어 세션 시작 요청에 포함되는가?
- [ ] 유효하지 않은 `documentId`로 진입 시 403/404 에러 처리 및 안내가 동작하는가?
- [ ] `documentId` 없이 진입 시 일반 면접으로 정상 시작되는가?

### 면접 이력 목록

- [ ] 면접 이력이 최신순으로 페이징 처리되어 정상 조회되는가?
- [ ] 빈 상태(Empty State): 면접 이력이 없을 때 안내 문구가 정상 노출되는가?
- [ ] 특정 이력 클릭 시 `sessionId` 기반으로 리포트 페이지로 재진입되는가?
