## Phase 1 — 상태 관리 & 데이터 연동

### 상태 설계

- [ ] 서버 상태(분석 결과, 이력 목록)는 `TanStack Query`로 관리하고, 입력 폼 상태는 컴포넌트 `useState`로 관리하는가?
- [ ] TanStack Query 캐시 데이터를 컴포넌트 `useState`에 복사하여 동기화를 깨뜨리지 않았는가?
- [ ] `idle → uploading → analyzing → success / error` UI 상태 전이가 누락 없이 처리되는가?

### 데이터 연동

- [ ] 데이터 계약: `api-schema.md`에 정의된 JSON 구조와 실제 API 호출/응답 구조가 일치하는가?
- [ ] 세션 유지: 페이지 새로고침 시에도 `documentId` 및 분석 결과가 유실되지 않는가?
  > SessionStorage 적용 (탭 종료 시 자동 만료 의도 — LocalStorage 사용 금지)
- [ ] 분석 중 페이지 이탈 후 재진입 시, 진행 중이던 상태가 올바르게 복원되는가?

### 에러 핸들링

- [ ] API 호출 실패 시 `ErrorCode`에 따른 공통 모달/메시지가 정상 노출되는가?
- [ ] 네트워크 단절 상태에서 업로드 시 적절한 에러 메시지가 표시되는가?

## Phase 2 — 기능 구현 (P1: 핵심 사용자 플로우)

> P1: 서류 분석의 핵심 플로우. 이 항목들이 동작하지 않으면 기능 자체가 불가.
>
> **입력 방식 구분**
> - 이력서: 파일 업로드 (PDF·DOC·DOCX)
> - 자기소개서: 텍스트 직접 입력 (문항 + 답변, 최대 5문항 / 문항당 1000자)

### 이력서 파일 업로드

- [ ] 용량 제한: 10MB 초과 파일 업로드 시 적절한 에러 메시지가 노출되는가?
- [ ] 확장자 필터링: PDF, DOC, DOCX 이외의 파일 업로드 시 차단 및 안내 메시지가 노출되는가?
- [ ] MIME 타입 검증: 확장자를 위장한 파일(예: `.pdf`로 위장한 `.exe`)을 클라이언트 측에서 차단하는가?
- [ ] 동일 파일 연속 업로드 시 중복 요청이 방지되는가? (debounce 또는 업로드 중 버튼 비활성화)

### 자기소개서 텍스트 입력

- [ ] 문항 추가: 최대 5문항까지만 추가 가능하며, 초과 시 추가 버튼이 비활성화되는가?
- [ ] 글자 수 제한: 문항당 1000자 초과 입력이 차단되고 현재 글자 수가 실시간으로 표시되는가?
- [ ] 지원 회사 / 지원 직무 필드가 비어 있을 때 분석 요청이 차단되고 안내 메시지가 노출되는가?
- [ ] 문항 또는 답변 중 하나만 입력된 상태에서 분석 요청 시 유효성 검사가 동작하는가?
- [ ] "AI 분석 시작하기" 버튼 클릭 시 중복 요청이 방지되는가? (요청 중 버튼 비활성화)

### 분석 로딩 Modal

- [ ] 분석 단계별(파일 읽기 → 키워드 추출 → 피드백 생성) 메시지가 동적으로 변경되는가?
- [ ] 분석이 비정상 종료(타임아웃, API 오류)될 경우 Modal이 닫히고 에러 처리가 되는가?

### 시각화 (Report)

- [ ] 종합 점수 차트(0 → N점) 애니메이션이 적용되었는가?
- [ ] 불필요한 리렌더링이 없는가? (React DevTools Profiler로 확인)

### 피드백 (Good / Bad / Fix)

- [ ] AI가 제공한 3단계 피드백 구조(Good/Bad/Fix)가 UI에 정확히 매핑되었는가?
- [ ] 피드백 항목 중 하나라도 비어 있을 때 빈 상태가 올바르게 처리되는가?

## Phase 3 — 연동 계약 (P2: 인접 도메인 연결)

> P2: 단독 기능은 동작하지만, 타 도메인과의 연결이 필요한 항목.

### 면접 도메인 연결

- [ ] "이 서류로 면접 시작하기" 버튼 클릭 시 `documentId`가 면접 도메인으로 전달되는가?
  > 전달 방식: `URL query parameter (?documentId=xxx)` — 다른 방식 사용 시 이 줄을 수정할 것
- [ ] 유효하지 않은 `documentId`로 면접 도메인 진입 시 에러 처리가 되어 있는가?

### 목록 관리

- [ ] 서류 분석 이력이 최신순으로 페이징 처리되어 정상 조회되는가?
- [ ] 빈 상태(Empty State): 분석 이력이 없을 때 안내 문구가 정상 노출되는가?
