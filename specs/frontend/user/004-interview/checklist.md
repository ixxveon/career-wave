# Checklist: Interview 도메인 (AI 실시간 모의 면접 - FE 집중)

> 작성자(이가연) 본인 검증용. 모든 항목 체크 후 PR 생성.

## Phase 1 — WebSocket 및 미디어 통신 (실시간성)
- [ ] WebSocket 연결/해제/재접속 로직 및 **네트워크 단절 시 복구 대기 UI** 구현
- [ ] 미디어 디바이스(카메라/마이크) 권한 획득 및 상태 감지(REC 아이콘 동기화)
- [ ] OpenVidu/WebRTC 세션 연동 및 미러링(`scaleX(-1)`) 적용 스트림 렌더링
- [ ] **소켓 이벤트 페이로드 명세(필드명·타입·nullable) 확인 및 타입 정의(TypeScript 등)**

## Phase 2 — 대화형 인터랙션 (UX)
- [ ] AI 질문 텍스트 타이핑 효과(Typing Animation) 구현
- [ ] 답변 제출 중 버튼 비활성화(중복 제출 방지) 및 로딩(Spinner) 표시
- [ ] **면접 타이머/질문별 제한 시간 UI 및 시간 초과 시 자동 제출 로직 구현**
- [ ] 대화 히스토리 자동 스크롤 및 입력창 [Enter/Shift+Enter] 정책 구현
- [ ] 면접 진행 상태(READY, RUNNING, FINISHED, REPORT)에 따른 UI 전환

## Phase 3 — 실시간 데이터 시각화 (성능 및 격리)
- [ ] 초당 반복되는 분석 데이터(WPM, 시선, 감정 등) 수신 및 리렌더링 최적화(memo/useMemo)
- [ ] 분석 지표(차트/게이지바) 데이터 바인딩 및 부드러운 애니메이션 적용
- [ ] 데이터 수신 빈도 제어(Throttling/Debouncing)를 통한 브라우저 성능 확보
- [ ] **Error Boundary를 활용한 분석 패널/채팅 영역 간 컴포넌트 에러 격리**

## Phase 4 — 리포트 및 결과 산출
- [ ] 면접 종료 후 종합 점수 도넛 차트 및 상세 피드백(아코디언) 렌더링
- [ ] AI 분석 피드백(오답/개선안) 및 핵심 포인트 텍스트 하이라이팅 적용
- [ ] 'PDF 리포트' 버튼 클릭 시 유저 등급별 조건부 모달(업그레이드 안내) 노출
- [ ] **결과 데이터 로컬 스토리지 보관 시 만료 정책 및 민감 데이터 필터링/보안 처리**

## Phase 5 — 코드 품질 및 테스트
- [ ] 상태 관리(Zustand/Recoil 등) 로직의 의존성 및 불필요한 리렌더링 확인
- [ ] 컴포넌트 구조 분리(VideoBox, ChatContainer, AnalysisPanel, ReportAccordion 등) 확인
- [ ] 반응형 레이아웃 확인 (질문 텍스트와 영상/채팅 영역의 가독성 확보)
- [ ] 콘솔 로그 및 테스트용 주석 제거

## 머지 전 최종 확인
- [ ] PR 제목 형식 준수: `[user-interview-기능명]`
- [ ] `tasks.md` 모든 작업 완료 체크 표시 확인
- [ ] **백엔드와 합의된 소켓 페이로드 스키마(필드·타입·nullable) 1:1 일치 확인**
