# Checklist: 서비스 통계 및 분석 (Statistics)

> `spec.md`가 "무엇을 만들지"라면, 이 파일은 "제대로 만들어졌는지" 검증한다.

---

## Phase 1 — UI 구현 검증

- [ ] KPI 카드 4종(이번 달 매출 / 누적 총 매출 / 총 가입자 수 / 이번 달 신규 가입)이 표시된다.
- [ ] 이번 달 매출 카드에 전월 대비 증감률이 표시된다.
- [ ] 이번 달 신규 가입 카드에 전월 대비 증감률이 표시된다.
- [ ] 월별 매출 추이 SVG 꺾은선 차트가 표시된다. (최근 6개월)
- [ ] 차트 피크 포인트에 최고 매출 툴팁이 표시된다.
- [ ] 구독 유형별 매출 실적 목록(프리미엄 월정액 / 신규 가입 전환 / 갱신 결제 / 환불 차감)이 표시된다.
- [ ] 각 유형별 매출액과 전월 대비 증감률이 표시된다.
- [ ] 구독자 변동 추이 SVG 이중 꺾은선 차트가 표시된다. (신규 vs 탈퇴, 최근 6개월)
- [ ] 차트 우상단 범례에 이번 달 신규/탈퇴 수가 표시된다.
- [ ] 최근 가입 피드 5건이 표시된다. (아바타 / 이름 / 상태 / 플랜 / 경과 시간)

---

## Phase 2 — API 연동 전 체크

- [ ] `StatsSummary` 인터페이스 필드가 백엔드 응답 DTO와 매핑 가능한지 확인한다.
- [ ] `MonthlyRevenue` 인터페이스 필드가 백엔드 응답 DTO와 매핑 가능한지 확인한다.
- [ ] `RevenueBreakdownItem` 인터페이스 필드가 백엔드 응답 DTO와 매핑 가능한지 확인한다.
- [ ] `MonthlySubscribers` 인터페이스 필드가 백엔드 응답 DTO와 매핑 가능한지 확인한다.
- [ ] `RecentSubscriber` 인터페이스 필드가 백엔드 응답 DTO와 매핑 가능한지 확인한다.
- [ ] SVG 차트의 `month` X축 레이블이 API 응답 `month("YYYY-MM")` 필드 기반으로 교체 가능한지 확인한다.
- [ ] 모든 HTTP 호출이 `statsApi.ts`를 통해서만 수행되는지 확인한다.
- [ ] API 성공·실패 응답이 `ApiResponse<T>` 형식을 기준으로 처리되는지 확인한다.
- [ ] API 호출 중 로딩 상태(스피너)가 표시되는지 확인한다.
