## Phase 1 — UI 구현

### KPI 카드

- [ ] KPI 카드 4종 렌더링 (전체 신고 수·접수(미처리)·블라인드 처리·기각 처리)

### 신고 목록 테이블

- [ ] 신고 목록 테이블 구성 (신고ID·신고 대상 유형·신고 사유·처리 상태·신고 일시·관리)
- [ ] 처리 상태 필터 드롭다운 (전체 / 접수 / 블라인드 / 기각)
- [ ] 신고 대상 유형 필터 드롭다운 (전체 / 게시글 / 댓글 / 회원)
- [ ] 신고 사유 필터 드롭다운 (전체 / 스팸 / 욕설·비방 / 광고·홍보 / 부적절한 콘텐츠 / 기타)
- [ ] 처리 상태 배지 (`PENDING → 접수`, `BLINDED → 블라인드`, `DISMISSED → 기각`)
- [ ] 신고 대상 유형 배지 (`BOARD → 게시글`, `COMMENT → 댓글`, `MEMBER → 회원`)
- [ ] 신고 사유 한글 매핑 상수 (`SPAM → 스팸`, `ABUSE → 욕설·비방`, `AD → 광고·홍보`, `INAPPROPRIATE → 부적절한 콘텐츠`, `OTHER → 기타`)
- [ ] 체크박스 UI (전체 선택·개별 선택) — 일괄 처리는 v2, UI만 구현
- [ ] 관리 컬럼 "상세보기" 버튼 (PENDING·BLINDED·DISMISSED 공통 노출)

### 신고 상세 모달

- [ ] 상세 모달 기본 정보 영역 (신고ID·신고 대상 유형·신고 사유·처리 상태·신고 일시)
- [ ] 신고자·피신고자 정보 영역
- [ ] 대상 콘텐츠 영역 (제목·본문 또는 댓글 내용)
- [ ] `BLINDED / DISMISSED` 상태 신고 — 처리 일시·처리 관리자 표시, 처리 버튼 비활성화 또는 미노출
- [ ] `PENDING` 상태 신고 — "블라인드 처리" / "기각 처리" 버튼 표시

### 블라인드 처리 확인 모달

- [ ] "확정" / "취소" 버튼
- [ ] 확정 버튼 클릭 시 블라인드 처리 API 호출

### 기각 처리 확인 모달

- [ ] "확정" / "취소" 버튼
