# Constitution: 신고관리 (Report Management)

**Feature Branch**: `feature/admin-report-ui`
**Scope**: 신고 목록 조회, 상세 모달, 블라인드·기각 처리, AI 검토, 회원 제재
**버전**: v1
**담당**: 신보라

---

## 1. 도메인 가치

관리자는 커뮤니티 게시글·댓글·회원에 대한 신고 내역을 한 화면에서 파악하고,
신고 콘텐츠의 블라인드·기각 처리, 콘텐츠 삭제, 회원 제재를 통해 플랫폼 품질을 유지할 수 있어야 한다.

AI 검토 의견은 관리자의 판단을 보조하는 참고 수단이며, 최종 결정은 관리자가 내린다.

---

## 2. 상태 머신

### 신고 처리 상태 전이 (report_status)

```text
PENDING
  → BLINDED   (관리자가 상세 모달에서 "블라인드" 버튼 클릭)
  → DISMISSED (관리자가 상세 모달에서 "기각" 버튼 클릭)

BLINDED
  → (변경 불가 — v1 기준)

DISMISSED
  → (변경 불가 — v1 기준)
```

| 전이 | 허용 여부 | 사유 |
|------|-----------|------|
| PENDING → BLINDED | 허용 | 상세 모달에서 단일 클릭으로 즉시 처리 |
| PENDING → DISMISSED | 허용 | 상세 모달에서 단일 클릭으로 즉시 처리 |
| BLINDED → 기타 | 불가 (v1) | 처리 버튼 미노출, API 409 반환 |
| DISMISSED → 기타 | 불가 (v1) | 처리 버튼 미노출, API 409 반환 |

## 4. 불변 규칙

- `BLINDED` 또는 `DISMISSED` 상태의 신고에 대해 처리 버튼을 노출하는 것을 금지한다.
- `target_type = MEMBER` 신고 블라인드 처리 시 "콘텐츠 블라인드" 안내를 표시하지 않는다. (신고 상태만 변경)
- AI 검토 결과는 관리자 판단을 보조하는 참고 수단이며, 자동으로 처리 상태를 변경하지 않는다.
- 신고 사유(`reason`) 한글 매핑은 프론트엔드 상수(`reasonLabel`)로 관리하며, UI에서 임의 변환하지 않는다.
- 처리 버튼 비활성화/미노출은 클라이언트 상태(`report_status`)만으로 판단하며, 서버 응답과 불일치 시 서버 응답을 우선한다.
- 회원 제재 처리 시 제재 사유 미입력 상태에서 "제재 적용" 버튼을 활성화하는 것을 금지한다.

---

## 5. 연동 계약

- `admin-backend`는 아래 API를 제공한다:
  - `GET /api/admin/reports/summary` — KPI 집계 (전체·처리 대기·블라인드 건수, AI 고위험 건수는 목록에서 클라이언트 집계 또는 별도 파라미터 추가 검토)
  - `GET /api/admin/reports?status=&targetType=&page=&size=` — 신고 목록 (페이지네이션)
  - `GET /api/admin/reports/{reportId}` — 신고 상세 (신고자·피신고자·대상 콘텐츠)
  - `PATCH /api/admin/reports/{reportId}/blind` — 블라인드 처리
  - `PATCH /api/admin/reports/{reportId}/dismiss` — 기각 처리
- 모든 응답은 `ApiResponse<T>` 형식 (`success`, `statusCode`, `message`, `data`)을 사용한다.
- 처리 성공 시 응답 `data`에 변경된 `reportStatus`가 포함되며, 이를 기반으로 목록 상태 뱃지를 즉시 갱신한다.

---

## 6. 금지 패턴

- `PENDING`이 아닌 신고의 처리 버튼을 노출하거나 활성화하는 것을 금지한다.
- AI 검토 결과를 근거로 자동 블라인드·기각 처리를 수행하는 것을 금지한다. (관리자 명시적 클릭 필수)
- 신고 처리 상태(`report_status`)를 서버 응답 없이 프론트엔드에서 임의로 확정하는 것을 금지한다.
- `target_type = MEMBER` 신고에 대해 "콘텐츠 블라인드" 안내를 표시하는 것을 금지한다.
- 회원 제재 모달에서 제재 사유 미입력 시 "제재 적용" 버튼을 활성화하는 것을 금지한다.
